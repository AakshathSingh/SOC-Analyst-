# SOC141 - Phishing URL Detected

## Executive Summary

On **March 22, 2021**, the Security Operations Center (SOC) received a **High Severity alert** for suspicious outbound communication from an internal endpoint, **EmilyComp (172.16.17.49)**. The investigation confirmed that the internal user **ellie** accessed a malicious phishing URL hosted on the domain **mogagrocol.ru**, which was later validated as malicious through threat intelligence sources.

The phishing URL contained a suspicious WordPress-style path and a victim-specific email parameter, indicating a targeted phishing attempt. Proxy and log analysis confirmed successful communication between the endpoint and the malicious infrastructure. External reputation analysis through VirusTotal further validated the threat, with multiple security vendors flagging both the URL and domain as malicious and phishing-related.

Since the request was **allowed** and not blocked by security controls, the endpoint posed a potential risk of credential theft, malicious payload delivery, or additional compromise. The affected system was immediately investigated and **contained** to prevent further malicious communication. This incident was classified as a **True Positive phishing event** requiring remediation and continued monitoring.

## Investigation and Analysis

### 1. Alert Investigation – Suspicious URL Detection

The investigation began with the review of the **SOC141 – Phishing URL Detected** alert generated by proxy monitoring systems. Initial telemetry identified suspicious outbound communication from the endpoint **EmilyComp (172.16.17.49)** to the domain **mogagrocol.ru**.

The alert provided multiple indicators commonly associated with phishing infrastructure:

* Communication to a suspicious **`.ru` domain**
* Use of **HTTP instead of HTTPS**
* A crafted URL path mimicking a **WordPress plugin directory**
* A **victim-specific email parameter** embedded in the request

The following URL was identified during the investigation:

```text
http://mogagrocol.ru/wp-content/plugins/akismet/fv/index.php?email=ellie@letsdefend.io
```

Evidence from the monitoring console confirmed:

* **Source Host:** EmilyComp
* **Source IP:** 172.16.17.49
* **Destination IP:** 91.189.114.8
* **User:** ellie
* **Request Status:** Allowed

### 2. Threat Intelligence Analysis – URL and Domain Validation

To validate the reputation of the identified indicators, the suspicious URL and domain were analyzed using **VirusTotal**.

#### URL Reputation Analysis

The phishing URL was submitted for reputation analysis and was flagged by **14/92 security vendors** as malicious.

The detections included:

* **Phishing**
* **Malicious**
* **Malware**

This provided strong evidence that the URL was associated with phishing activity.

#### Domain Reputation Analysis

The domain **mogagrocol.ru** was separately analyzed and flagged by **12/91 security vendors** as malicious.

Threat intelligence classified the domain under:

* **Phishing and Fraud**
* **Malicious Infrastructure**

These findings increased confidence that the endpoint communicated with confirmed malicious infrastructure.

## Conclusion and Recommendations

The **SOC141 – Phishing URL Detected** alert was determined to be a **True Positive phishing incident** involving successful communication with malicious infrastructure.

The internal endpoint **EmilyComp (172.16.17.49)** accessed the phishing domain **mogagrocol.ru (91.189.114.8)** through a suspicious HTTP request containing victim-specific tracking parameters. Threat intelligence validation confirmed that both the URL and domain were malicious and phishing-related.

Since the request was **allowed**, the endpoint was exposed to potential phishing threats, including credential theft and malicious payload delivery. As a precautionary measure, the endpoint was investigated and contained.

### Recommendations:

#### CONTAINMENT – Maintain Device Isolation

Maintain isolation of **EmilyComp (172.16.17.49)** until a complete endpoint review confirms no malicious persistence or indicators of compromise remain.

#### BLOCK INDICATORS

Block the following indicators at perimeter security devices and web filtering controls:

* **Domain:** `mogagrocol.ru`
* **IP Address:** `91.189.114.8`
* **Associated URL patterns**

#### INVESTIGATION

Conduct a complete endpoint review to identify:

* Browser credential theft
* Downloaded malicious files
* Persistence mechanisms
* Suspicious child processes

#### USER AWARENESS

Provide phishing awareness training to reduce the likelihood of future phishing-related incidents.

#### SECURITY IMPROVEMENT

Strengthen web filtering policies to automatically block malicious domains and suspicious HTTP-based phishing attempts.
