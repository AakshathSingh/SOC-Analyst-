# SOC170 – Passwd Found in Requested URL – Possible LFI Attack

## Executive Summary

On **March 01, 2022**, the Security Operations Center (SOC) received a **High Severity alert** indicating a possible **Local File Inclusion (LFI) / Directory Traversal attack** against the web server **WebServer1006 (172.16.17.13)**.

Investigation revealed that an external host (**106.55.45.162**) attempted to access the sensitive Linux file **/etc/passwd** through a manipulated URL parameter using directory traversal sequences (`../../../../`). Such attacks are commonly used by threat actors to access files outside the web application's intended directory structure.

Log analysis confirmed that the malicious request reached the target web server; however, the server responded with **HTTP Status Code 500 (Internal Server Error)** and returned **0 bytes** of data. No evidence was found indicating successful disclosure of sensitive information or compromise of the server.

Threat intelligence enrichment through VirusTotal did not identify the source IP as malicious. Nevertheless, the request itself clearly demonstrated malicious intent and matched known LFI/Directory Traversal attack patterns.

The incident was classified as a **True Positive** representing an **unsuccessful exploitation attempt**.

# Alert Investigation

The investigation began by reviewing the alert generated by the monitoring platform.

The alert was triggered because the requested URL contained a reference to the Linux passwd file:

```http
https://172.16.17.13/?file=../../../../etc/passwd
```

The payload contains multiple directory traversal sequences (`../`) designed to move outside the application's intended directory structure and access sensitive files on the underlying operating system.

The targeted file:

```bash
/etc/passwd
```

is a common target during reconnaissance and exploitation attempts because it contains user account information on Linux systems.

### Key Findings

* External IP attempted access to a sensitive system file.
* URL contained a directory traversal payload.
* Request was allowed to reach the web server.
* Alert correctly identified malicious behavior.

# Raw Log Analysis

Further validation was performed through examination of the raw web logs.

The following information was identified:

```text
Request URL:
https://172.16.17.13/?file=../../../../etc/passwd

Request Method: GET
Device Action: Permitted
HTTP Response Size: 0
HTTP Response Status: 500
```

### Analysis

The HTTP response code returned by the server was:

```text
500 Internal Server Error
```

A successful LFI attack would typically return:

```text
200 OK
```

along with the contents of the requested file.

Because:

* Response Size = 0 Bytes
* HTTP Status = 500
* No file content was returned

the attack was determined to be unsuccessful.

# Conclusion

The **SOC170 – Passwd Found in Requested URL – Possible LFI Attack** alert was determined to be a **True Positive**.

Investigation confirmed an external actor attempted to exploit a potential Local File Inclusion vulnerability by requesting the Linux system file **/etc/passwd** through a directory traversal payload.

Although the request successfully reached the web server, the server returned **HTTP 500 Internal Server Error** with a response size of **0 bytes**, indicating the attack was unsuccessful and no sensitive information was disclosed.

No evidence of system compromise was identified, and Tier 2 escalation was not required.

---

# Recommendations

### Web Application Hardening

* Validate and sanitize all user-supplied file path parameters.
* Implement strict allow-listing for file access functionality.

### Deploy WAF Protections

Block known traversal patterns:

```text
../
../../
../../../../etc/passwd
```

### Continuous Monitoring

Monitor for:

* LFI attempts
* Directory Traversal payloads
* Sensitive file access requests
* Repeated exploitation attempts from external hosts

### Security Testing

Perform regular:

* Vulnerability Assessments
* Web Application Penetration Testing
* Secure Code Reviews

to identify and remediate file inclusion vulnerabilities before exploitation occurs.
