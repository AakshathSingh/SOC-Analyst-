# SOC137 - Malicious File/Script Download Attempt

## Executive Summary

On **March 07, 2021**, the Security Operations Center (SOC) received a **Medium Severity alert** for a **Malicious File/Script Download Attempt** involving endpoint **NicolasPRD (172.16.17.37)**. The investigation revealed that a suspicious **macro-enabled Microsoft Word document** (`INVOICE PACKAGE LINK TO DOWNLOAD.docm`) initiated malicious activity through **PowerShell execution** and outbound communication to suspicious external infrastructure.

Endpoint telemetry and network logs confirmed that **PowerShell (`powershell.exe`)**, launched via **Windows Management Instrumentation (`wmiprvse.exe`)**, attempted outbound communication with suspicious domains over **HTTP (80)** and **HTTPS (443)**. Additional threat intelligence validation through **VirusTotal** confirmed malicious behavior consistent with **trojan/downloader activity**, including macro execution, PowerShell abuse, and external payload retrieval attempts.

Although endpoint controls partially mitigated the activity, suspicious outbound communication was successfully observed. The endpoint was immediately **contained** to prevent further malicious activity or secondary compromise. This incident was classified as a **True Positive malicious script execution and downloader activity attempt**.

## Investigation and Analysis

The investigation began with the review of the **SOC137 – Malicious File/Script Download Attempt** alert generated by endpoint monitoring systems. Initial telemetry identified suspicious activity associated with a **macro-enabled Microsoft Word document** named:

```text
INVOICE PACKAGE LINK TO DOWNLOAD.docm
```

The filename strongly suggested a **social engineering lure**, commonly used in phishing or malware delivery campaigns involving fake invoices or package notifications.

* Macro-enabled document (`.docm`)
* Suspicious PowerShell execution
* External network communication
* Downloader/trojan-related behavior

Evidence from the monitoring console confirmed:

* **Source Host:** NicolasPRD

### 3. Raw Log Analysis – PowerShell Execution Confirmation

To validate malicious execution behavior, raw logs associated with the suspicious connections were analyzed.

The logs confirmed successful outbound communication to the suspicious domain:

```text
iluuryeqa.info
```

The following process chain was identified:

* **Process:** `powershell.exe`
* **Parent Process:** `wmiprvse.exe`
* **Request Method:** GET
* **Device Action:** Allowed

The parent-child process relationship is highly suspicious because **WMIPRVSE.EXE spawning PowerShell** is frequently associated with:

* Malware downloaders
* Script-based execution
* Living-off-the-Land techniques
* Defense evasion

This evidence strongly suggests that the malicious document attempted to execute scripts and retrieve additional payloads.

## Conclusion and Recommendations

The **SOC137 – Malicious File/Script Download Attempt** alert was determined to be a **True Positive malware incident** involving a suspicious **macro-enabled Microsoft Word document** designed to execute malicious scripts and retrieve external payloads.

Investigation confirmed suspicious **PowerShell execution**, **outbound communication to suspicious domains**, and **threat intelligence indicators** consistent with trojan/downloader behavior. VirusTotal analysis validated the malicious nature of the document, while endpoint telemetry confirmed suspicious process execution and network communication.

Although endpoint controls reduced the impact of the activity, evidence confirmed malicious behavior occurred and warranted immediate containment.

### Recommendations:

#### CONTAINMENT – Maintain Device Isolation

Maintain isolation of **NicolasPRD (172.16.17.37)** until a complete forensic review confirms no persistence or additional indicators of compromise remain.

#### BLOCK INDICATORS

Block the following indicators at perimeter security controls:

**Domains**

* `iluuryeqa.info`
* `ueba6ka.club`

**IP Addresses**

* `49.51.12.195`
* `31.214.157.60`

#### INVESTIGATION

Conduct a complete endpoint review to identify:

* Scheduled tasks
* Registry persistence
* Downloaded payloads
* Suspicious child processes
* Credential theft artifacts
* WMI abuse

#### USER AWARENESS

Provide user awareness training focused on:

* Invoice-themed phishing documents
* Macro-enabled Office attachments (`.docm`)
* Malware delivery via malicious attachments

#### SECURITY IMPROVEMENT

Strengthen endpoint protections through:

* Office macro restrictions
* PowerShell logging
* Script block logging
* WMI monitoring
* Application allowlisting
