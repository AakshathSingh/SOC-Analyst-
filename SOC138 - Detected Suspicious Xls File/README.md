# SOC138 - Detected Suspicious XLS File

## Executive Summary

On **March 13, 2021**, the Security Operations Center (SOC) received a **Medium Severity alert** for a **Suspicious XLS File Detection** involving endpoint **Sofia (172.16.17.56)**. The investigation revealed that a suspicious **macro-enabled Microsoft Excel document** (`ORDER SHEET & SPEC.xlsm`) initiated malicious activity through **macro execution**, suspicious outbound communication, and attempted interaction with malicious external infrastructure.

Endpoint telemetry and network logs confirmed that the malicious Excel file attempted communication with suspicious infrastructure over **HTTPS (443)**. Additional raw log analysis identified communication with the suspicious URL:

```text
http://stylefix.co/guillotine-cross/CTRNOQ/
```

Further threat intelligence validation through **VirusTotal** confirmed malicious behavior consistent with **trojan/downloader activity**, including **macro execution, obfuscated behavior, malicious payload delivery techniques, and downloader functionality**.

The endpoint security solution did **not quarantine the file**, as endpoint telemetry confirmed the action status as **Allowed**, increasing the likelihood of successful execution. Due to confirmed malicious communication and validated malware behavior, the endpoint was immediately **contained** to prevent additional compromise or secondary payload execution.

This incident was classified as a **True Positive malware infection involving a malicious macro-enabled XLSM file**.

## Investigation and Analysis

The investigation began with the review of the **SOC138 – Detected Suspicious XLS File** alert generated by endpoint monitoring systems. Initial telemetry identified suspicious activity associated with a **macro-enabled Microsoft Excel document** named:

```text
ORDER SHEET & SPEC.xlsm
```

The use of an **`.xlsm` extension** strongly indicated the presence of **embedded VBA macros**, a common malware delivery mechanism frequently abused in phishing campaigns and malicious document-based attacks.

The alert revealed several indicators commonly associated with malware delivery and execution:

* Macro-enabled Excel file (`.xlsm`)
* Suspicious outbound network communication
* Downloader/trojan-related behavior
* Malicious URL interaction
* Endpoint security action marked as **Allowed**

Evidence from the monitoring console confirmed:

* **Source Host:** Sofia
* **Source IP:** 172.16.17.56
* **User:** Sofia2020
* **Threat Category:** Trojan / Downloader / Dropper

### 2. Log Analysis – Suspicious Network Communication

To determine whether the malicious Excel document initiated outbound communication, **Log Management telemetry** was reviewed.

The logs confirmed suspicious outbound communication from endpoint **172.16.17.56** to suspicious external infrastructure over **HTTPS (443)**.

Evidence showed:

| Timestamp               | Destination Address | Port | Connection Type |
| ----------------------- | ------------------- | ---- | --------------- |
| Mar 13, 2021 – 08:20 PM | 177.53.143.89       | 443  | Firewall        |

The communication to an unknown external IP immediately after execution strongly suggested **command-and-control (C2)** or **payload retrieval behavior**, commonly associated with downloader malware.

This activity significantly increased confidence that the macro-enabled document executed malicious instructions and attempted external communication.

## Conclusion and Recommendations

The **SOC138 – Detected Suspicious XLS File** alert was determined to be a **True Positive malware incident** involving a malicious **macro-enabled Excel document (`.xlsm`)** designed to execute malicious actions and communicate with suspicious external infrastructure.

Investigation confirmed suspicious **outbound communication**, malicious **macro behavior**, and external communication with suspicious infrastructure. Threat intelligence validation through **VirusTotal** confirmed trojan/downloader characteristics, while endpoint telemetry confirmed the file execution was **Allowed**, increasing the likelihood of successful malicious execution.

Although endpoint containment successfully limited further activity, the evidence strongly suggests the malicious document executed and attempted communication with external infrastructure.

### Recommendations

#### CONTAINMENT – Maintain Device Isolation

Maintain isolation of **Sofia (172.16.17.56)** until a complete forensic investigation confirms no persistence or secondary payloads remain.

#### BLOCK INDICATORS

Block the following indicators at perimeter security controls:

**Domain**

* `stylefix.co`

**IP Address**

* `177.53.143.89`

**URL**

* `http://stylefix.co/guillotine-cross/CTRNOQ/`

#### INVESTIGATION

Conduct a complete endpoint review to identify:

* Scheduled tasks
* Registry persistence
* Downloaded payloads
* Suspicious child processes
* Credential theft indicators
* Macro execution artifacts

#### USER AWARENESS

#### SECURITY IMPROVEMENT
